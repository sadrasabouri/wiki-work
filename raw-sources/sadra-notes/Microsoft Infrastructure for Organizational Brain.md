+ Work IQ [[microsoftwork-iq MCP Server and CLI for accessing Work IQ]]: mostly similar to what we already have [[workiq]]; the main idea is an agent that connects to your personal tools (calendar, slack, ...) and let you interact with the data
+ Fabric IQ: This is a knowledge graph that business owners create over their data sources to see the dependencies between organizational structures. It's for managers to get an ideas between semantic relations that different entities have between each other. They call the knowledge graph ontology and they call connected components "Lake." Being a single source of truth for company. AI agents can interact with this ontology via APIs MCPs. Important note is that the relations are constructed between entities that are already existing as an object and the relations that carry (limited to) predefined edge type. Users can publish their chats with others the agent that sees the data lakes (views but not updated but they can run it anyways, called agent muscles [[FabricIQ How are you using AI capabilities to bring value to the business?]]). There's concept of publishing the chat as an "operational agent" which is similar to view but it's mostly like saying people to use my prompts (like skills) https://www.youtube.com/watch?v=37LHUPvT1gU.
+ Foundry IQ: Integrating organizational knowledge such as documents, policies, manuals, FAQs, contracts, wikis, and other unstructured information [[What is Foundry IQ?]].
This post ([[Demystifying Microsoft’s AI Strategy Work IQ, Fabric IQ, Foundry IQ, and More]]) summarizes these three concepts

The differences I see between [[wiki-kb-project]] and these systems are two main idea (which are both powered by agents)
1. Utilizing transcripts information to inform Foundry IQ; how enriched that could be. before agents we could not extract meaningful information from it, now we can
2. Lost connection between personal knowledge and organization knowledge. We want to track individuals to contribute to organization knowledge (reverse link that FoundryIQ + WorkIQ + FabricIQ has)
