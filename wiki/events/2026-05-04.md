# May 4 — Two Sumit-Sadra 1:1s: DAG, Living System, Alternative Foundations

Sources: [[Sadra-4May2026-TRANSCRIPT2]], [[Sadra-4May2026-TRANSCRIPT3]]

Two unplanned late-night syncs (2AM and 6AM). Sumit called because ideas were occurring to him; both are short and dense.

## Full-System Provenance DAG (TRANSCRIPT2, 13min)

Sumit extended the [[dag-of-computation|DAG of Computation]] concept from the spreadsheet context to the entire wiki system. The proposal: a graphical view showing computational dependencies across ALL levels — raw data → wiki entities → view outputs — so users see what was computed from what. "So that the user knows what exists and what was computed from what."

Key clarifications:
- **Wiki is just another node.** "Wikis are also a special kind of view." In the full system DAG, raw data, wiki entities, and view outputs are all nodes at different refinement levels. Edges are computational relationships.
- **Multiple sources per node.** A wiki entity may be sourced from multiple raw files; a view may source from multiple wiki entities. The graph is a multi-graph.
- **AI slop flag.** AI-created nodes should carry a human-validation flag. This connects to [[provenance-tracking]].

This view is itself a view — one more slash command Sadra can add.

## Temporal Navigation via Source Filtering (TRANSCRIPT2)

Sumit proposed a mechanism for temporal navigation orthogonal to [[temporal-knowledge|purge/supersede]]: let users filter which input sources participate in computing a view. Default = all sources. Filter out recent transcripts = simulate past state. "The moment we give them provenance to select which objects should flow... they can simulate the temporal aspect of things."

This makes temporal navigation a side effect of provenance selection, not a separate caching architecture. See [[temporal-knowledge]] for how this relates to the existing purge mechanism.

## Automated Raw Data Generation (TRANSCRIPT3, 20min)

Sumit challenged the assumption that raw data is always manually ingested. The process that *generates* raw data can itself be a creative workflow captured in the system: "get top 5 Twitter posts with at least 100 likes, ordered by recency." When this process is rerun, the raw data updates. The user's creativity is in specifying the data-gathering criteria, not just in specifying what to do with the data. See [[living-system]].

## Everything is a DAG — Clean Formulation (TRANSCRIPT3)

Sumit gave the cleanest formulation of the system's architecture: **"Nodes are data, edges is a workflow. That's it."** Every artifact in the system — Twitter post, wiki concept page, view output — is just data at some refinement level. Every transformation between them is a workflow. The entire system is a DAG.

A view = prompt (declarative spec) + optional workflow (constructive spec of HOW to compute it). When only a prompt exists, the LLM figures out the workflow. When a workflow is specified, it constrains and accelerates computation.

## Wiki as Indexing Infrastructure (TRANSCRIPT3)

Sadra articulated why wiki is a *special* kind of node in the DAG: "the interesting thing about wiki being a view is that it provides an indexing mechanism." Wiki doesn't just compute a result — it creates the index that makes future views efficient. This is the justification for wiki having privileged status over other intermediate views.

## Alternative Foundational Views (TRANSCRIPT3)

Sadra proposed two alternative infrastructure views that could complement or replace wiki as the indexing layer:

1. **Knowledge graph** — typed edges (source → relation → target); more interpretable, probably smaller graph; each edge has a semantic label (e.g., "person owns concept")
2. **Event log / database** — structured events extracted from raw data; optimized for temporal queries

These are open proposals, not implemented. They arose from the observation that wiki might not be the only valid indexing intermediate.

## AI-AI Transcripts (TRANSCRIPT3)

Sumit floated spinning up two AI agents to discuss a topic and using the resulting transcript as raw data. Sadra resisted — skeptical of current AI's ability to generate genuinely insightful content. Sumit clarified the intent: it would be clearly labeled (not mixed with human-human data), and filters would handle it differently. Left open as a future possibility.
